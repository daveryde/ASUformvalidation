Validating Submitted Data

By: David Cordova

Introduction

JavaScript is commonly used with forms for two reasons: to add functionality that makes forms easier for users to fill out, and to validate or process the data that a user enters before that data is submitted to server-side script. Validation is the process of checking that information provided by users conforms to rules to ensure that it appropriately answers the form's questions and is provided in a format that the site's back-end programs can work with.

In Chapters 1 - 3, we learned core JavaScript concepts, accessing the DOM, and events. In Chapters 4 - 6, we learned how to manipulate the DOM, debug, and implement form validation. One event, submit, is available for use with the form element. The submit event fires when a form is submitted, which is generally when a submit button is selected on a form. In this article, we will briefly discuss browser based form validation and custom JavaScript form validation. There are many ways to pick a DOM node that lets us access specific elements, such as a form, but we will target the click event for full custom validation.

Browser Based Validation

Form validation normally used to occur at the server, after the client entered all the necessary data and then pressed the submit button. JavaScript provides a way to validate form's data on the client's computer before sending it to the web server. HTML5 has introduced features to validate form input. Some examples are checking for empty fields when you requires text input or checking if a number was typed into a number input and not a string. The form will display validation errors (bubbles) near the input that did not pass validation.

For basic form validation, HTML5 integrates client-side validation by providing attributes that can be placed into your HTML. This makes validation faster and easier to implement, however, it's not ideal if your project involves checking for specific input. Here are some of the new atttributes available for your form.

JavaScript Validation

Form validation can be handled two ways: along side the contraint validation API to customize aspects of the way browsers present browser-based validation or handled entirely by custom JavaScript code. We are going to review custom JavaScript form validation starting with our main function to control or handle the steps we want our validation to go through.

A. event.preventDefault(); disables the default behavior for the referenced event, essentially blocking the action normally associated with the submit event.

B. retrieveElements(); accesses the DOM and find our form elements once the client submits the form.

C. createListeners(); will listen for the submit button to be clicked.

D. validateForm(); is where we write our custom validation code based on the values submitted by the client.

1. Form Controller

function handleForm(event) {
  /* Prevent form submit */
  event.preventDefault();

  /* Initialize elements */
  retrieveElements();

  /* Setup event listeners */
  createListeners();

  /* Validate Form */
  validateForm();
}

Code The Validation

The form validation rules are written via comparing the client submitted values against a test case scenario. JavaScript has built-in functions that make checking values a bit easier. In validateForm(), we use the built-in function includes('@') to check if the string value of email included the '@' symbol.

To keep this article brief, there are different ways we can compare a value using the built-in functions but make sure you understand the type of value you are validating. The built-in functions may give unexpected results when you are unsure of the value type (i.e. string, integer, boolean).

BAD:

Checking whether a string variable name is a number 

var name = 'David';

/* "isNaN" means Not-A-Number.
   "!isNaN" means the opposite of Not-A-Number
   This checks whether the variable 'name' is a number */
if(!isNaN(name)) {
  messageDiv.textContent = 'Looks Good!';
}

GOOD:

Checking whether an integer variable number is a number 

var number = 12345;

if(!isNaN(number)) {
  messageDiv.textContent = 'Looks Good!';
}

Test / Debug

When writing JavaScript code, you will most likely run into bugs or issues with your code. The first step to tracing and debugging your code is to put console.log(variable); statements in your code, monitor the output in your browser's console, and adjust accordingly. If this doesn't help and you need additional debugging support, then you may access your browser developer tools. Chrome, Firefox, and Edge provide in depth documentation on how to use these to debug your code.

'use strict';

/* Global variables for user input element */
var email,
  password,

function handleForm(event) {
  /* Prevent form submit */
  event.preventDefault();

  /* Initialize elements */
  retrieveElements();

  createListeners();

  /* Validate Form */
  validateForm();
}

function retrieveElements() {
  /* Get form input elements from DOM */
  email = document.querySelector('#inputEmail');
  password = document.querySelector('#inputPassword');

  city= document.querySelector('#inputCity');
  state = document.querySelector('#inputState');
  zip = document.querySelector('#inputZip');
  checkBox = document.querySelector('#inputCheck');
  radio = document.querySelector('#inputRadio');
}

function createListeners() {
  /* Find the submit button and listen for click event */
  submit = document.getElementById('submit');
  submit.addEventListener('click', handleForm, false);
}

function validateForm() {
  /* Check input and display custom message */
  if (email.value === '' || !email.value.includes('@')) {
    emailFeedback.classList.add('text-danger');

    email.classList.remove('border', 'border-danger');

/* Create an event listener when the DOM loads */
if (window.addEventListener) {
